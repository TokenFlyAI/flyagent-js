# 🎯 Task Planning Demo

A fully browser-native task planning app where **the entire UI is generated by an AI agent as Markdown** with embedded interactive widgets.

**No React. No Vue. No Svelte. No traditional UI code.**

Just:
- 🧠 **Kimi LLM** for intelligence
- 📄 **Markdown** for UI description
- 🎛️ **markdown-ui.com DSL** for interactive widgets
- 🚀 **flyagent-js** for the agent framework

---

## ✨ Features

1. **Mission Creation** — describe your project, the AI sets up the plan
2. **AI-Suggested Tasks** — agent analyzes your mission and proposes tasks with estimates
3. **Timeline Setup** — deadlines and durations auto-calculated
4. **Progress Visualization** — live pie charts, bar charts, and line charts
5. **Track & Control** — update status, priority, and progress via interactive widgets

---

## 🏗️ Architecture

```
┌─────────────┐     Markdown      ┌─────────────────┐
│   Kimi LLM  │ ────────────────► │  Browser Page   │
│  (Agent)    │ ◄──────────────── │  (Renderer)     │
└─────────────┘   Widget Events   └─────────────────┘
```

1. The AI agent generates Markdown with embedded `markdown-ui-widget` code blocks
2. A vanilla JS renderer parses the widgets and creates interactive DOM elements
3. User interactions (clicks, sliders, form submits) are sent back to the agent as text
4. The agent updates its internal state and regenerates the Markdown
5. The UI updates in real-time

## 🚀 Quick Start

### 1. Get a Kimi API Key

Sign up at [platform.moonshot.cn](https://platform.moonshot.cn) and create an API key.

### 2. Build flyagent-js (if not already built)

```bash
cd ../..
npm run build
```

### 3. Build the demo

```bash
cd demos/task-planning
npm run build
```

### 4. Open in browser

```bash
# Using Python's built-in server
python3 -m http.server 8080

# Or using Node.js
npx serve .
```

Then open `http://localhost:8080` in your browser.

### 5. Enter your API key and click **Start**

## 🎨 Widget Types Supported

| Widget | Syntax | Use Case |
|--------|--------|----------|
| `text-input` | `text-input id "Label" "placeholder" "default"` | Mission name, descriptions |
| `button-group` | `button-group id ["A" "B"] "default"` | Task status selection |

| `slider` | `slider id 0 100 1 50` | Progress percentage |
| `form` | `form id "Submit"` + indented children | Create mission, add task |
| `chart-pie` | `chart-pie` + CSV data | Overall completion |
| `chart-bar` | `chart-bar` + CSV data | Tasks by priority |
| `chart-line` | `chart-line` + CSV data | Timeline/burndown |

## 🧠 How It Works

### The System Prompt

The agent is given a detailed system prompt that:
- Defines the task plan data model (mission, tasks, status, priority, progress)
- Teaches the markdown-ui widget DSL syntax
- Specifies 5 behavior modes (mission creation → task suggestion → active planning → timeline → tracking)
- Instructs the agent to persist state via `savePlan()` / `loadPlan()` tools

### The Event Loop

```
1. User opens page → agent generates mission creation form
2. User fills form → event captured → "User interacted with widget 'createMission': {...}"
3. Agent receives event → analyzes → generates new markdown (task suggestions)
4. Renderer updates the DOM
5. User clicks "Accept All" → event captured → agent transitions to Active Planning mode
6. Agent renders full task board with charts
7. User drags slider on Task 3 → agent updates progress → recalculates charts → re-renders
```

## 🛠️ Customization

### Change the LLM model

Edit `app.ts` and change the model string:

```typescript
new Agent({
  model: 'moonshot-v1-32k',  // or 'gpt-4o', 'gemini-1.5-pro'
  // ...
})
```

### Add more widget types

Edit `renderer.ts` and add a new case to `renderWidget()`:

```typescript
case 'date-picker': return renderDatePicker(config);
```

### Change the color theme

Edit the `<style>` block in `index.html`.
